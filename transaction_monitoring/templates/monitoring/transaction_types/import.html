CSV Import

CSV File Format

Your CSV file should have the following columns:

  • transaction_code * - Unique identifier (e.g., CSH+)
  • description * - Human-readable description
  • jurisdiction (optional) - Country code (e.g., USA)
  • groups (optional) - Comma-separated list of group codes (e.g., CCE-INN,ALL-ALL)

Example:

transaction_code,description,jurisdiction,groups
CSH+,Cash Deposit,,CCE-INN,ALL-ALL
CSH-,Cash Withdrawal,,CCE-OUT,ALL-ALL
WIRE_INT+,International Wire Transfer In,USA,WIRE-INN,ALL-ALL

If a group code doesn't exist, it will be automatically created.

Understanding Transaction Types

Transaction Type System

The transaction type system is a core component of AML monitoring. It categorizes financial transactions to allow rules to target specific activities.

Important Concepts

  • Transaction Codes: Unique identifiers for specific transaction types (e.g., CSH+, WIRE_INT+)
  • Transaction Groups: Collections of related transaction types (e.g., CCE-INN for all cash equivalent deposits)
  • Group Hierarchy: Groups can have parent-child relationships (e.g., ALL-ALL contains all transaction types)

How It's Used in Rules

When configuring AML rules, you can specify which transaction types the rule should monitor:

  • Rules can target specific transaction codes for precise monitoring
  • Rules can target transaction groups to monitor categories of activity
  • The rule code itself often indicates the transaction groups it monitors (e.g., AML-LCT-CCE-INN-P-D01-EST)
