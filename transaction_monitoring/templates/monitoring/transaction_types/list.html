About Transaction Types

Transaction types are specific codes that identify the nature of financial transactions in the system. They are used in AML rules to target specific transaction activities for monitoring.

Examples include:

  • CSH+: Cash deposit
  • CSH-: Cash withdrawal
  • CSH_CP+: Cash deposit at teller

Transaction types are organized into groups (e.g., "CCE-INN" for Cash Equivalent-Incoming) to make rule configuration easier. When a rule is configured with a transaction type group, it will monitor all transactions with types in that group.
