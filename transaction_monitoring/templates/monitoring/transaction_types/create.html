Transaction Type Format

Transaction types typically follow a specific format indicating the transaction nature and direction:

  • Format: [TYPE]_[SUBTYPE]_[LOCATION][DIRECTION]
  • Direction: + for incoming (deposits), - for outgoing (withdrawals)
  • Examples: CSH+ (cash deposit), CSH- (cash withdrawal), WIRE_DOM+ (domestic wire transfer in)

Each transaction type should be assigned to one or more transaction type groups which are used in rule configurations.
