Transaction Type Group Naming

Transaction type groups typically follow a structured naming convention that indicates the category and direction of transactions:

  • Format: [CATEGORY]-[DIRECTION]
  • Examples:
    • CCE-INN: Cash and Cash Equivalent Incoming transactions
    • WIRE-OUT: Wire Transfer Outgoing transactions
    • ALL-ALL: All transaction types (both incoming and outgoing)

Parent-child relationships can be established to create a hierarchy of transaction types, allowing for more granular rule targeting.

How Groups Are Used

Purpose of Transaction Type Groups

Transaction type groups serve several important purposes in your AML monitoring system:

Rule Targeting

Groups allow rules to target specific categories of transactions without having to list every individual transaction code. For example, a rule can monitor all cash deposits by targeting the CCE-INN group.

Hierarchical Organization

Parent-child relationships between groups enable both broad and specific monitoring. A rule could target ALL-INN (all incoming) or more specifically CCE-INN (cash equivalent incoming).

Rule Code Structure

Groups are embedded in rule codes to show what transactions they monitor. For example, AML-LCT-CCE-INN-P-D01-EST includes CCE-INN to indicate it monitors cash equivalent incoming transactions.

Maintainability

When transaction codes change or new ones are added, you can simply update the group membership rather than modifying individual rules.
